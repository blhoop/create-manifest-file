# Output Format — Spoke Infrastructure Manifest (v1.3.0)

Defines the structure of the `.yml` file generated by this application.
Schema reference lives in `schema-template-v1.3.0.md`.

---

## File Naming

```
{spoke-name}.yml
```

---

## Structure Overview

```
# <spoke-name> -- Spoke Infrastructure Manifest
# Schema: manifest-schema.json v1
schema_version: '1.6.0'

spoke:             — identity & metadata (from subscription panel)
ctm_properties:    — resource naming product code
environments:      — target environment + location
tags:              — applied to all resources
network:           — vnets, subnets, NSGs, private endpoints, DNS zones
compute:           — plan_defaults, web_apps, function_apps, static_sites,
                     container_app_environment, container_apps, aks_clusters,
                     virtual_machines, signalr, apim
data:              — databases, storage_accounts, caching, search, factories,
                     backup_vaults, messaging
security:          — key_vaults, container_registries, managed_identities
app_configuration: — centralized app settings
ai:                — AI Foundry (foundry[])
frontdoor:         — Front Door profile + routing
observability:     — log_analytics_workspace, app_insights
dependencies:      — external spoke connections
```

## Spoke Panel Fields

Filled in via the UI subscription panel. Grouped into three sections.

### Identity (required)

| Field | Description |
|-------|-------------|
| `spoke_name` | Unique spoke identifier — used for repo names, TFC workspaces, and the YAML header |
| `owner` | Team that owns this spoke |
| `product` | Short product code used in all resource names (2–5 chars, e.g. `lb`) |
| `environment` | `dev` `test` `uat` `preprod` `prod` `lab` |
| `default_location` | Azure region for all resources |

### Tagging

| Field | Description |
|-------|-------------|
| `cost_center` | For chargeback/showback |
| `project` | Project name |
| `data_classification` | `internal` `confidential` `public` `restricted` |
| `infra_repo` | GitHub repo name for generated Terraform |

### Infrastructure

| Field | Description |
|-------|-------------|
| `sku_mode` | `premium` (default) or `standard` |
| `vnet_cidr` | VNet address space — triggers VNet integration on compute resources |
| `management_group_id` | Short MG ID (e.g. `converge`) |
| `new_subscription` | `true` = LZ creates subscription / `false` = use existing |
| `subscription_id` | Existing subscription UUID (shown only when `new_subscription = false`) |
| `description` | Human-readable workload description |

## Resource Row Fields

Each row in the preview table maps to one resource entry.

| Field | Required | Description |
|-------|----------|-------------|
| `name` | ✅ | Subsystem/component name — used in resource naming |
| `type` | ✅ | Resource type (see table below) |
| `location` | | Azure region override — omit to use `default_location` |
| `repo` | | SCM repo (`org/repo-name`) — noted as `app_repo` comment in output |
| `comments` | | Structured `Key:Value` pairs set via popup — drives OS, SKU, runtime, etc. |

## Plan Defaults

`app_service_plan` rows drive `compute.plan_defaults` rather than appearing as explicit plan entries:

- **First web ASP row** (or `PlanFor: Web App`) → `plan_defaults.web_app`
- **First func ASP row** (or `PlanFor: Function App`) → `plan_defaults.function_app`
- **Additional ASP rows** → `plan_override` on the matching app (matched by name)

Set `PlanFor`, `OS`, and `SKU` on each ASP row via its comment popup.

---

## NSG Rules & Consumers

Set via dedicated tabs in the resource comment popup:

- **NSG Rules tab** — available on `web_app`, `app_service`, `app_service_plan`, `function_app`. Rules are collected, deduplicated, and emitted under `network.nsgs.nsg_appservices.rules`.
- **Consumers tab** — available on `key_vault`. Emitted as `security.key_vaults[].consumers`.

---

## Network Auto-Generation

When compute rows (web apps / function apps) are present:
- `network.subnets` includes `snet_appservices` with `Microsoft.Web/serverFarms` delegation
- `network.nsgs` includes `nsg_appservices`
- Each web/function app gets `vnet_integration_subnet_id: snet_appservices` (when `vnet_cidr` is set)

Private endpoints and DNS zones are DERIVED by the builder — omit or add manual overrides only.
